
# Google AI Overview Remover

Removes Google Search's "AI Overview" (Gemini / AI blocks) from search results. This extension targets AI Overview blocks whether they appear at the very top of the page or later in the search results (including nested or dynamically-inserted variants).

## Quick summary

- Detects AI Overview blocks using multiple signals (hidden headings, visible labels, data attributes, common class names and text patterns).
- Observes dynamic changes with a debounced MutationObserver that watches child nodes, attributes and character data.
- Removes the detected AI block or the nearest enclosing search-result container to ensure nested blocks are removed.
- Includes conservative protections to avoid removing essential page UI (search box, headers, RHS knowledge panels, etc.).

## Files

- `content.js` — main content script (detection, removal logic, observer). Modified to handle AI blocks inserted later in the results.
- `content-debug.js` — debug version with verbose logging (useful to capture DOM patterns when diagnosing misses).
- `background.js` — background service worker that reinjects the content script on navigation.
- `manifest.json` — manifest V3 configuration.
- `test.html`, `test_extension.html` — local test pages (if present) to exercise the content script.

## Install (Load unpacked)

1. Open Chrome (or a Chromium-based browser) and go to `chrome://extensions/`.
2. Enable "Developer mode" (top-right).
3. Click "Load unpacked" and select this extension folder:

   `d:\Computer\Codings\Extensions\Google AI Overview Remover`

4. Ensure the extension is enabled.

Note: `manifest.json` is already configured to run the content script on Google Search URLs.

## How to test (manual)

1. Open DevTools (F12) → Console on a Google Search page.
2. Run a query that triggers an AI Overview (example used in testing: `pakistan gdp`).
3. Watch the Console for messages such as:
   - `Google AI Overview Remover: Removed X AI Overview elements`
   - `Google AI Overview Remover: Aggressively removed element for selector ...`
   - `Google AI Overview Remover: Removed AI Overview inside changed node`

4. Try interactions that may insert the block later: scroll, click a result and go back, paginate, or change the query. The enhanced observer detects attribute/characterData updates and added nodes.

If you still see the AI Overview:

- Right-click the AI block → Inspect → copy outer HTML (Right-click node → Copy → Copy outerHTML) and paste it into an issue or message. That allows adding a precise selector.

## Debugging options

- To enable verbose debug output temporarily, either:
  - Change the manifest's content script list to use `content-debug.js` instead of `content.js`, or
  - Replace `content.js` with `content-debug.js` (keep a backup) for local debugging, then reload the extension.

- With debug mode active, examine the Console to see what nodes and attributes the extension is matching.

## Detection details (high level)

- Primary signals used by `isAIOverview(element)`:
  - Hidden accessibility headings (clipped `h1`/`h2`) containing "AI overview".
  - Visible label elements such as `.Fzsovc` and other Google-specific label classes.
  - Attributes like `data-mcpr`, `data-mg-cp`, `data-mcp`, `data-ved` containing `ai`, `jscontroller` values, or `data-ai` flags.
  - Text patterns that mention AI (e.g., "AI Overview", "AI-generated", "generated by AI").

- Removal approach:
  1. Quick scan of known selectors and remove nearest result/container.
  2. Scan standard search-result containers (e.g. `.g`, `.tF2Cxc`, `.MjjYud`, `.g-blk`) and their descendants for AI markers.
  3. Fallback limited scan over `div`, `section`, `article`, `main` with a cap to avoid heavy DOM work.
  4. Aggressive pass explicitly targets variants like `div.YzCcne[data-mcpr]`, `.Fzsovc`, and clipped headings.

## Safety & false positives

The extension includes `isCriticalElement()` checks to avoid removing core UI. However, broad AI-related text patterns can sometimes match legitimate results that mention "AI".

If you see false positives:

1. Copy the offending block's outer HTML and open an issue with the HTML and the search query used.
2. The easiest mitigation is to refine the detection patterns (e.g., require both label + hidden heading) or add a specific selector for that variant.

## Developer notes

- To add a selector: open `content.js`, find `quickSelectors`, `aiOverviewSelectors` or `aggressiveSelectors` and add a new selector. Keep changes conservative and test.
- The MutationObserver is debounced and limits per-run removals to avoid thrashing and reduce risk of accidental mass removals.
